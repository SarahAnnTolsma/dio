# Inlining Transformers

## ProxyFunctionInliningTransformer

**Status: Stub** - not yet implemented.

Inlines proxy functions generated by JavaScript obfuscators. These are small functions that wrap a single binary operation, forward a call, or return their argument unchanged.

Runs in the **Main** phase with **First** priority so that inlined expressions are exposed to downstream transformers (constant folding, control flow simplification, etc.).

### Binary operation proxy

```js
// Before
function _0x1(a, b) { return a + b; }
_0x1(1, 2);

// After (planned)
1 + 2;
```

### Call forwarding proxy

```js
// Before
function _0x4(a, b) { return a(b); }
_0x4(fn, arg);

// After (planned)
fn(arg);
```

### Identity proxy

```js
// Before
function _0x5(a) { return a; }
_0x5(x);

// After (planned)
x;
```

### Planned algorithm

1. Scan for function declarations whose body is a single return statement
2. Classify the proxy kind (binary, call forwarding, identity)
3. Verify via scoping that the function is only used as a direct callee
4. Inline each call site using `operations::replace_expression`
5. Remove the proxy declaration (dead code elimination handles this)
