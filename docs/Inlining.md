# Inlining Transformers

## ProxyFunctionInliningTransformer

Inlines proxy functions generated by JavaScript obfuscators. These are small functions that wrap a single binary operation, forward a call, or return their argument unchanged. The function declaration is removed after all call sites are inlined.

Runs in the **Main** phase with **First** priority so that inlined expressions are exposed to downstream transformers (constant folding, control flow simplification, etc.).

### Binary operation proxy

```js
// Before
function _0x1(a, b) { return a + b; }
var x = _0x1(1, 2);

// After
var x = 1 + 2;
// (then constant folding simplifies to: var x = 3;)
```

### Call forwarding proxy

```js
// Before
function _0x4(fn, a) { return fn(a); }
_0x4(alert, "hi");

// After
alert("hi");
```

### Identity proxy

```js
// Before
function _0x5(a) { return a; }
var x = _0x5(42);

// After
var x = 42;
```

### Multiple call sites

All call sites for a proxy function are inlined:

```js
// Before
function _0x1(a, b) { return a + b; }
var x = _0x1(1, 2);
var y = _0x1(3, 4);

// After
var x = 3;
var y = 7;
```

### Safety checks

The transformer only inlines functions that:
- Have a body consisting of a single return statement
- Reference only their own parameters in the return expression (no free variables)
- Are not async or generator functions
- Have no rest parameters
- Are never reassigned (no write references to the function binding)
